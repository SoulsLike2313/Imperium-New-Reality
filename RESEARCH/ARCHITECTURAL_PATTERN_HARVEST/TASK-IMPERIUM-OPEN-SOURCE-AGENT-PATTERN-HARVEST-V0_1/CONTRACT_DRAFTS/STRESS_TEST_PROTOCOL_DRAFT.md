# Stress-Test Protocol Draft

## Purpose

Every Organ-Agent must be tested against success, rejection, risk, and performance cases.

## Required test classes

1. Good input accepted.
2. Missing provenance rejected.
3. Forbidden path blocked.
4. Direct canon mutation refused.
5. Runtime output classified as runtime.
6. Receipt written.
7. Report output budget respected.
8. CLI plain JSON works.
9. CLI pretty mode works without hiding errors.
10. Dirty tree not hidden.
11. Route recommendation correct.
12. Side-effect tool locked or rejected when unsafe.
