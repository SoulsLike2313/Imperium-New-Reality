# Permission / Admission Protocol Draft

## Purpose

Define how Organ-Agents decide whether an action is allowed.

## Admission stages

1. Input shape validation.
2. Path scope validation.
3. Agent authority validation.
4. Tool verification validation.
5. Side-effect classification.
6. Custodes admission for risky actions.
7. Owner approval for high-risk actions.
8. Receipt requirement.

## High-risk actions

- delete files
- mutate canon
- cross-zone merge
- external tool installation
- credential access
- private/local context intake
- parallel stateful agent execution
- network access
- unbounded repo scans
