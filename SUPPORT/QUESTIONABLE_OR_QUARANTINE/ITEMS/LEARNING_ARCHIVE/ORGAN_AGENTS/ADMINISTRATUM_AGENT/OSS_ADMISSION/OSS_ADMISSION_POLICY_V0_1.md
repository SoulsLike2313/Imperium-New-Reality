# OSS Admission Policy V0.1

## Core rules
- detect availability only
- no auto-install
- no silent hard dependency introduction
- every tool requires purpose/scope/risk/fallback

## Admission flow
1. candidate registration
2. local availability detection
3. safety and fallback validation
4. owner review
5. optional admission update

## Output discipline
- machine JSON register remains canonical
- owner report can summarize in human form
