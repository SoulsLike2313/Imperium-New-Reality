# Frontend Skill Spine V0.1

## Purpose
Standardize frontend architecture behavior for New Generation while preserving truth-binding discipline.

## Must-cover capability
- folder/component architecture boundaries;
- UI primitives vs feature widgets split;
- accessibility baseline and reduced motion respect;
- state ownership boundaries;
- styling and token separation;
- performance awareness without fake performance claims.

## Skill passport candidates
1. `frontend.structure_guard.v0_1`
2. `frontend.ui_primitive_vs_widget_split.v0_1`
3. `frontend.accessibility_baseline_guard.v0_1`
4. `frontend.state_boundary_mapper.v0_1`
5. `frontend.token_layer_integrator.v0_1`
6. `frontend.visual_truth_binding_checker.v0_1`
7. `frontend.evidence_screenshot_mapper.v0_1`

## Step protocol baseline
1. Confirm visual objective and truth source mapping.
2. Split primitives, domain widgets, and app shell responsibilities.
3. Keep tokens/motion/textures in reusable layers.
4. Validate reduced-motion and keyboard-readable behavior.
5. Emit screenshot/report evidence with path references.

## Forbidden moves
- mixing domain business logic into primitive components;
- hard-coding colors/motion where tokenized layer is required;
- claiming UX/performance pass without evidence artifacts;
- treating reference image as direct implementation contract.

## Evidence expectations
- component and folder mapping note;
- token-to-surface mapping;
- accessibility checks or explicit limitation;
- screenshot/report receipt linkage.
