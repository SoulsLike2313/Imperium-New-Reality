# Backend Skill Spine V0.1

## Purpose
Create backend service/contract discipline for New Generation without fake integration claims.

## Must-cover capability
- service boundaries and ownership;
- contracts and schemas as first-class artifacts;
- API and adapter separation;
- receipts/reports as operational evidence;
- persistence candidates and data integrity notes;
- no fake live backend claims.

## Skill passport candidates
1. `backend.service_boundary_planner.v0_1`
2. `backend.contract_first_schema_guard.v0_1`
3. `backend.api_adapter_split.v0_1`
4. `backend.receipt_generation_contract.v0_1`
5. `backend.persistence_candidate_assessor.v0_1`
6. `backend.no_fake_live_integration_guard.v0_1`
7. `backend.error_surface_and_risk_mapper.v0_1`

## Step protocol baseline
1. Define service responsibility and boundary.
2. Bind inputs/outputs to schema contracts.
3. Isolate adapter layer from domain logic.
4. Attach receipt/report generation path.
5. Record unresolved runtime dependencies as explicit unknowns.

## Forbidden moves
- route or API claims without contract documentation;
- passing mocked outputs as live system proof;
- mixing report/evidence artifacts inside service source roots;
- claiming persistence safety without declared constraints.

## Evidence expectations
- schema references;
- route/adapter mapping table;
- known failure and fallback notes;
- receipt-ready artifact path list.
