# Visual Foundry Skill Spine V0.1

## Purpose
Codify visual topology work so visual quality evolves with truth-binding and evidence discipline.

## Must-cover capability
- visual topology and address contracts;
- visual unit passports;
- token layer and texture layer separation;
- motion layer intent and safety;
- screenshot evidence discipline;
- reference-to-implementation mapping;
- Playwright/evidence candidate path for future tasks.

## Step protocol baseline
1. Map target surface to visual address and truth source.
2. Define unit passport fields before UI mutation.
3. Route style through token/texture/motion layers.
4. Capture before/after screenshots and attach to receipts.
5. Record unknowns and placeholders explicitly.

## Forbidden moves
- declaring visual readiness without truth-map linkage;
- treating decorative polish as semantic system completion;
- claiming performance pass without measurements;
- moving from concept sketch to finality claim without staged evidence.

## Evidence expectations
- visual address registry references;
- proof screenshots index;
- mapping from reference intent to implementation decision;
- explicit list of unresolved visual contract items.
