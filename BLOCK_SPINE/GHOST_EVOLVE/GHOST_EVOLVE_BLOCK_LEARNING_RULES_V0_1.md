# GHOST_EVOLVE_BLOCK_LEARNING_RULES_V0_1

Status: CANDIDATE_NOT_CANON
Owner organ: SCHOLA_IMPERIALIS
Support organs: STRATEGIUM, INQUISITION, MECHANICUS

## Learning lifecycle

1. Record pain memory with source evidence paths.
2. Record success memory with measurable delta.
3. Open structured improvement request.
4. Link request to owning block and organ.
5. Define script-first remediation target.
6. Re-evaluate with cap state transition evidence.

## Required fields per learning item

- learning_id
- source_problem
- evidence_paths
- owning_block
- affected_organ
- cap_mapping
- prevention_rule
- script_first_remediation
- expected_useful_delta
- observed_useful_delta
- status

## Efficiency delta rule

Efficiency delta must be measured from at least one explicit before and after metric.
No clean pass claim is allowed when delta is missing or negative.
