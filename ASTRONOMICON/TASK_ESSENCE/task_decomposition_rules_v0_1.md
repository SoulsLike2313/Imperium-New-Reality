# Task Decomposition Rules V0.1

1. Start from one explicit task essence.
2. Split into compact phases with clear deliverables.
3. Bind each phase to receipts/evidence.
4. Validate after each phase (json parse, python compile, checker run, scope check).
5. Do not merge core docs, validation, and report output into one giant step.
6. Keep route packet executable by Servitor and future WARP template layer.
