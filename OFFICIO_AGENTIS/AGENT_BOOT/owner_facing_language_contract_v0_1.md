# Owner-Facing Language Contract V0.1

## Rule

Owner-facing live comments and explanations must be Russian.

## Allowed English Surface

English is allowed for:

- code;
- JSON/JSONL keys;
- schemas;
- logs;
- file paths;
- CLI commands;
- machine-facing receipts.

## Violation Handling

If owner-facing live commentary drifts into English:

1. Correct immediately in the next message.
2. Continue owner-facing commentary in Russian.
3. Record the correction note in final report if drift occurred.

## Final Summary Rule

Final owner-facing summary/comments must also be Russian.

## Reason

This contract reduces owner interpretation friction and enforces consistent communication discipline at runtime.
