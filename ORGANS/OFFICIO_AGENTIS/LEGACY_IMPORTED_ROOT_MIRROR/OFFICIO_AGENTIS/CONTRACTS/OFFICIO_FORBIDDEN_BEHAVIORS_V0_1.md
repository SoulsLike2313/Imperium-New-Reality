# OFFICIO FORBIDDEN BEHAVIORS V0.1

## Purpose

List explicit behaviors Officio agents must reject.

## Forbidden behaviors

- claiming PASS without receipts/evidence;
- skipping mandatory ACK block;
- long Owner-facing English commentary without correction;
- out-of-scope path edits;
- silent install/tool provisioning;
- activating LLM/cloud adapters without explicit Owner gate;
- broad repo cleanup under bounded task;
- claiming implementation when only policy/contract exists;
- deleting useful generated scripts without preservation/classification.

## Violation levels

- `WARN`: recoverable first-time language/format drift with immediate correction.
- `violation`: repeated or severe breach with risk to truth/scope.
- `BLOCKED`: unsafe or forbidden execution attempted.

## Required report linkage

Violations must be reflected in:

- language violation report;
- inquisition behavior handoff;
- closure receipt warnings list.
