# Review Reinforcement Proposals Contract V0.1

Status: CANDIDATE_V0_1
Owner organ: INQUISITION

## Purpose

Every Inquisition review must leave exactly three reinforcement proposals so the review loop improves New Reality instead of only blocking claims.

## Required Proposal Set

Each review must produce exactly one proposal of each type:

1. skill_api_tool_candidate
2. matrix_gate_improvement
3. cost_context_output_optimization

## Required Fields

Each proposal must include:

- proposal_id
- type
- why_now
- first_use_case
- risk
- admission_mode

## Admission Rule

Proposals are candidates, not canon. Any tool, API, or skill proposal must enter Mechanicus candidate storage and can become canon only after 10 successful task cycles with receipts.

## Verdict Rule

An Inquisition review is incomplete if it lacks exactly three reinforcement proposals or if two proposals use the same type.
