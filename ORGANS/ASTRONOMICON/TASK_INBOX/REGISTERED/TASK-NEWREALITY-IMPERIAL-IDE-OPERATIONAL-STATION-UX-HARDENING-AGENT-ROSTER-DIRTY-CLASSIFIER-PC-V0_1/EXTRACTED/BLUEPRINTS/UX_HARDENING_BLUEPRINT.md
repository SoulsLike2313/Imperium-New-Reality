# UX HARDENING BLUEPRINT

## Principle

The Owner sees human summaries first and raw JSON only on demand.

## Required screen pattern

Every operational screen should have:

- title;
- status;
- one-line human meaning;
- key paths;
- next recommended action;
- raw JSON available as secondary view;
- copy/open actions or copy-ready commands.

## Priority screens

- Dashboard;
- Agents and Servitors;
- Taskpack Manager;
- Launch Card;
- Handoff Card;
- Reports;
- Receipts;
- Dirty Classifier;
- Safety Center;
- Git Closure;
- Lifecycle.

## Do not

- do not make truncated JSON the only UI;
- do not hide blockers;
- do not display candidate as canon;
- do not display handoff-ready as execution-done.
