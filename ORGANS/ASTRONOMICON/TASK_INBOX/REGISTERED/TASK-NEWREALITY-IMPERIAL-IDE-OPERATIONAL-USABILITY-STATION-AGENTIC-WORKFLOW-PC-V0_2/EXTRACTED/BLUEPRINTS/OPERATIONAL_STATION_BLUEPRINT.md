# OPERATIONAL STATION BLUEPRINT

## Goal

Build one coherent operational station over the existing Imperial IDE layers.

The Owner must be able to work from inside the station.

## Primary surfaces

- Workbench GUI is the visual cockpit.
- Workbench TUI is the preferred fast operator surface.
- Imperial IDE CLI remains the scriptable surface.

## Required workflow

Owner intent
-> Task Console
-> MetaOS route preview
-> Mechanicus policy check
-> Taskpack Builder
-> Astronomicon local registration gate
-> Launch Card
-> Servitor Prime handoff
-> Lifecycle tracker
-> Reports and receipts
-> Git closure

## Non-negotiable rule

Do not replace the liked Workbench TUI with a weaker separate TUI.

Upgrade it in place or add a station mode reachable from it.
