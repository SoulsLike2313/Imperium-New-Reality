# Ghost_Evolve Block Spine Teaching Rules

Every block, context, IDE, or tool-surface issue must become durable system learning.

Examples:
- Servitor had to read too many files -> create context pack economy rule.
- Organ does not explain itself compactly -> organ passport improvement request.
- Protected zone is unclear -> protected zone contract update.
- Tool candidate lacks risk or resource budget -> Mechanicus candidate gate.
- IDE requirement is too broad -> split into workbench, surface, resource governor, and tool chamber contracts.
- External API candidate is unsafe -> Inquisition risk cap.
- Routine code validation consumes LLM tokens -> Mechanicus local validator candidate.

Each learning item must include:
- source problem;
- evidence path;
- owning block;
- affected organ;
- cap mapping;
- future prevention;
- script-first remediation path;
- expected useful delta.
