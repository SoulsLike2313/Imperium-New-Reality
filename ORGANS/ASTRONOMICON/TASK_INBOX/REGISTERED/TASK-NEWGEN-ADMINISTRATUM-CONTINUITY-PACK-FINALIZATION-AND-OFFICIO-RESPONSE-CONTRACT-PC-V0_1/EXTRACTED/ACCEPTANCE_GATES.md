# Acceptance gates

## Gate A: scope lock

PASS only if the task remains PC-contour and does not implement IDE, WARP, VM2/VM3 live route proof, browser runtime, external API runtime, freelance readiness, or trading readiness.

## Gate B: Administratum continuity finalization

PASS only if fresh finalization artifacts are created or updated under the Administratum report/skill area and include:

- current git truth;
- current accepted HEAD;
- current task chain;
- active caps;
- stale-context audit;
- private-data exclusion receipt;
- source index;
- fresh Logos-Prime handoff RU/EN runtime outputs or receipts;
- SHA256 or equivalent integrity evidence for the continuity package if a ZIP is built.

## Gate C: Officio response contract

PASS only if an Officio-owned contract exists or is updated with the exact non-BLOCK final response format:

1. Step name.
2. Step verdict.
3. Commit links with short labels for each task commit.
4. Exactly 3-4 short Russian Owner-facing comment lines.

The contract must say that detailed evidence belongs in repo/report/bundle, not chat.

## Gate D: Doctrinarium git closure law

PASS only if a Doctrinarium-owned law exists or is updated requiring non-BLOCK task results to be committed and pushed unless a specific approved exception applies.

## Gate E: Inquisition guard

PASS only if an Inquisition receipt or guard record states how future audits should flag:

- final response format drift;
- missing commit links;
- stale commit_push_receipt;
- unpushed non-BLOCK result;
- fake green closure claims.

## Gate F: no private leak

PASS only if private/local/secrets content is not copied into repo artifacts. If local/private paths are referenced, they must be described as excluded or allowlist-only.

## Gate G: git closure

PASS only if the final non-BLOCK result is committed, pushed, and `origin/master == HEAD` is verified. The task must produce a fresh `commit_push_receipt.json`.

## Gate H: final response compliance

PASS only if the final chat answer follows the exact 4-part response contract.
