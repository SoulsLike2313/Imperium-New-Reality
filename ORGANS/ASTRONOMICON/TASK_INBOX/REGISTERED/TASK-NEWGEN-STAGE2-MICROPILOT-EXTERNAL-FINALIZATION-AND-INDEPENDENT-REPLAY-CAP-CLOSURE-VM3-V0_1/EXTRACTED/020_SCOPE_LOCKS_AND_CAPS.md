# Scope locks and caps

## Caps targeted for closure or narrowing
- CAP_EXTERNAL_FINALIZATION_RECEIPT_MISSING_OR_NEEDS_FOLLOWUP
- CAP_NO_LOCAL_INDEPENDENT_REPLAY_IN_REVIEW_ENVIRONMENT

## Caps to carry transparently
- CAP_STAGE1_WITH_WARNINGS_ONLY
- CAP_NO_IDE_VISUAL_RELEASE_YET
- CAP_NO_WARP_RUNTIME
- CAP_DIRTY_START_OWNER_APPROVED_CONTINUATION

## Strict locks
- No IDE runtime.
- No WARP runtime.
- No browser automation runtime.
- No external API runtime.
- No AdsPower runtime actions.
- No freelance readiness claim.
- No trading readiness claim.
- No broad repo reading as default execution mode.

## Claim discipline
If a cap cannot be closed, record `NARROWED`, `CARRIED`, or `BLOCKED` with evidence and next required proof. Do not force a clean PASS.
