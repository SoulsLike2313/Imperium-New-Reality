# Ancient Empire and New Reality Epoch Policy

## Ancient Empire

Ancient Empire is the old full Imperium world preserved as precedent memory. It teaches but it does not execute active New Reality work.

It may contain old experiments, runtime artifacts, previous architecture forms, useful examples and mistakes. It must not be treated as active truth by default.

## New Reality

New Reality is the active runtime core. It contains organs, skills, matrices, contracts, reports, task registration and future Throne/Custodes integration.

## Epoch formula

```text
ANCIENT_EMPIRE teaches.
NEW_REALITY acts.
THRONE remembers.
CUSTODES admits.
```

## Comparison

Future comparison tasks may compare Ancient Empire and New Reality by:

- task closure quality;
- context cost;
- evidence quality;
- fake-green reduction;
- skill reuse;
- owner manual burden;
- organ participation quality;
- development dynamics.
