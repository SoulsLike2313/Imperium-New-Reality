## Required concepts

### Bundle as historical delta

The bundle is not just a report. It is the historical delta of a task:

- taskpack reference
- route manifest
- entry head
- post-task local head
- remote head after push
- changed file index
- receipt index
- organ receipt index
- validation receipts
- repair loop receipts
- local heavy artifact references
- GitHub-safe index cards
- next task route
- Russian owner summary after Officio role entry

### GitHub as index, not dump

Do not push large local-heavy bundles by default. GitHub should receive indexes and receipts:

- bundle index card
- receipt index
- file delta index
- final owner summary
- closure receipts
- hashes and local vault references

Large screenshots, runtime dumps, private context, heavy archives, or local-only evidence must be indexed, hashed, and classified instead of blindly committed.

### Nine-organ post-work summary

The ring must define or prepare receipts for:

- ASTRONOMICON: task flow and sequence
- OFFICIO_AGENTIS: role, language, behavior
- ADMINISTRATUM: bundle assembly and index
- MECHANICUS: tool and script delta
- DOCTRINARIUM: law, scope, doctrine
- INQUISITION: dirty state, fake green, contradiction scan
- STRATEGIUM: micro and macro delta, KPD where measurable
- SCHOLA_IMPERIALIS: learning, pattern memory, enhanced ghost evolve
- CUSTODES: narrow audit of organ receipts and matrix compliance

Custodes is not full Throne scope. It is only a narrow matrix auditor for this task.
