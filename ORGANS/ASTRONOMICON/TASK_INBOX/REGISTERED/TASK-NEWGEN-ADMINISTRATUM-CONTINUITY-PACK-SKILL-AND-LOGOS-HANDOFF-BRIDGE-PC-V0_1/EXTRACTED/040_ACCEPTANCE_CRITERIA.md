# Acceptance criteria

PASS_WITH_WARNINGS is acceptable if Stage/IDE/WARP caps remain carried.

A valid result requires:

- Administratum Skill package exists under Administratum ownership.
- A continuity pack can be built on PC by a short command or callable Skill entrypoint.
- The continuity ZIP contains a manifest, current truth snapshot, git truth receipt, latest task chain, active caps, source index, private-data exclusion receipt, stale-context audit, Logos-Prime handoff summaries, SHA256 checks and ZIP receipt.
- The Skill does not dump the whole repo.
- The Skill does not collect private/secrets.
- Owner-facing Russian handoff exists only as runtime/output artifact under Officio authority.
- Internal canonical files are English-only UTF8 no BOM.
- Inquisition guard explicitly states whether continuity is fresh, stale, partial or blocked.
- Mechanicus validation receipt covers JSON parse, ZIP readability and SHA256 generation.
- The task records that Astronomicon Taskpack Registration Skill was used for PC registration if observable.
- Commit/push receipt is fresh if commit/push is performed.

Clean PASS is not allowed if Stage1/IDE/WARP global caps remain active.
