# Servitor Control Chain Requirements

This task must define a model-agnostic control chain.

The future Servitor may be:

- Codex
- Claude
- Qwen
- Kimi
- Gemini
- a local LLM
- another CLI or API agent

The control must come from IMPERIUM organs, not from ad-hoc Owner chat corrections.

Required control chain:

1. Astronomicon gives task route and launch context.
2. Officio gives role, runtime language, response, and behavior contract.
3. Doctrinarium gives non-negotiable law.
4. Administratum provides current truth and continuity.
5. Mechanicus provides tool/validator capability map.
6. Inquisition provides compliance and no-fake-green matrix.
7. Speculum provides architecture red-team where needed.
8. Schola records lessons and reusable fixtures.

Required receipt fields:

- control_contract_issued_by_officio
- launch_context_injected_by_astronomicon
- doctrinarium_law_read
- inquisition_compliance_check_run
- observed_owner_facing_language
- required_owner_facing_language
- drift_detected
- correction_source
- agent_control_failure
- next_control_fix
