# Acceptance Gates

## Hard BLOCK gates

The task must stop or report BLOCK if any of these occur:

- New Reality root cannot be resolved.
- The active root is not `E:/IMPERIUM_NEW_GENERATION_NEW_REALITY`.
- Ancient Empire is mutated.
- Files are permanently deleted.
- External tools are installed without explicit Mechanicus admission.
- Work is pushed to the Ancient Empire repository.
- Worktree cannot be made clean after final closure.
- Remote HEAD cannot be verified after push.
- Report bundle is missing.
- Bundle SHA256 is missing.
- Required receipts are missing from report folder or evidence index.
- Machine/canonical artifacts contain Cyrillic without explicit Officio owner-runtime localization exception.
- Servitor control is corrected manually by Owner and the failure is not recorded as an agent-control failure.

## PASS conditions

A local step can pass only if:

- New Reality root resolution passes.
- Ancient Empire is not mutated.
- Repo classification is produced.
- Quarantine moves are hash-backed and reversible.
- Servitor control chain contract exists.
- Mechanicus candidate storage exists and has seeded cards.
- Evidence index/final proof standard exists.
- Reinforcement proposal contracts exist.
- Validators run and produce receipts.
- Commit and push complete to New Reality remote.
- `origin/master == HEAD`.
- Worktree is clean.
- Final bundle path and SHA256 are provided.

## Accepted warning conditions

Warnings may remain if:

- Self-reference limitation is disclosed.
- Some files are classified as `CANDIDATE_REVIEW` instead of moved.
- A candidate tool is registered but not installed.
- Some legacy report references remain in reports/history, not runtime paths.

## Forbidden fake-green cases

Do not claim clean pass if:

- A final commit is only visible in screenshot but not in receipts.
- A bundle is claimed but path/SHA256 are absent.
- A validator ran only in prepare phase but not verify phase.
- Owner manually corrected Servitor behavior and this is not disclosed.
- A full repository archive is used as substitute for task evidence bundle.
