# Skill Terminology And Ownership

Use the term Skill as the primary architecture term.

A Skill is a reusable, validated agent capability. It is an operational hand of an organ or agent. It can be called by TUI, CLI, IDE, local agent or Servitor, but its truth source remains with the owning organ.

For this task:

- Skill name: Astronomicon Taskpack Registration Skill.
- Owner organ: Astronomicon.
- Future caller: IDE, Sanctum, local agents, Servitor.
- Non-owner callers must not mutate truth directly.

Avoid canonizing the word launcher. It may remain as a compatibility alias for scripts or user convenience, but the architectural term is Skill.
