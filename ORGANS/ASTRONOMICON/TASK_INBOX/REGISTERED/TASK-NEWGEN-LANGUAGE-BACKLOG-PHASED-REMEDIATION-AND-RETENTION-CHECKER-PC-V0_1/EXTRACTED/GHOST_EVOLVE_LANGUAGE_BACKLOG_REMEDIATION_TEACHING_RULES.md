# Ghost_Evolve Language Backlog Remediation Teaching Rules

Every remediation or retention issue must become durable system learning.

Examples:
- A backlog item is safe to fix -> create a deterministic remediation rule.
- A backlog item is too risky -> leave it in backlog with owner organ and decision reason.
- Registered taskpack footprint is growing -> retention checker rule.
- EXTRACTED payload is large -> hash-and-quarantine candidate.
- Strict scope stays clean but global scan warns -> no clean PASS and phased plan.
- A false positive is found -> update filter signature logic.

Each learning item must include:
- source problem;
- evidence path;
- owning organ;
- affected route;
- cap mapping;
- future prevention;
- script-first remediation path.
