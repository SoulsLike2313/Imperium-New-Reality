# Acceptance Gates

## PASS_WITH_WARNINGS requires all

- VM3 starting repo truth recorded.
- VM3 starts from expected HEAD or records safe sync to current origin/master.
- VM3 worktree clean before task.
- Astronomicon intake on VM3 registers this taskpack.
- TASK_ID resolver works on VM3.
- Servitor starts from TASK_ID plus start task, not from raw long prompt.
- AGENTS, Matrix Spine, and required organ participation packets are acknowledged.
- 8-organ route receipt exists.
- VM3 route readiness snapshot exists.
- Next pipeline handoff points to block/context optimization, not IDE first.
- No block/context optimization implementation is performed in this route-proof task.
- No IDE, WARP, API arsenal, freelance, or trading work starts.
- Hard red-team verdict exists and can downgrade.
- Efficiency delta receipt exists.
- Commit and push are performed for admitted changes.
- Worktree is clean after push.
- Remote origin/master equals final HEAD.

## BLOCK

- VM3 repo cannot sync to origin/master.
- VM3 worktree is dirty before task and cannot be classified safely.
- Astronomicon intake fails and is not recorded.
- TASK_ID resolver fails and no proof is produced.
- Servitor executes only from raw prompt without task_id route.
- 8-organ route receipt is missing.
- Useful VM3 readiness snapshot is missing.
- Next task jumps to IDE/API arsenal without block/context optimization proof.
- Final report contains PENDING commit or push fields while claiming pass.
- No positive efficiency delta.
- Commit or push fails.
- Worktree remains dirty after finalization.
