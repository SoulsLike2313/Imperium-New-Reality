# DOCTRINARIUM Servitor Contract V0.1

## Input Contract

Servitor query must include:
- `task_id`
- `question`
- optional `mode`

## Output Contract

Return `newgen_organ_verdict_v0_1` with:
- scoped verdict;
- applied doctrinal rules;
- required and forbidden actions;
- evidence requirements;
- not-proven boundary.

## BLOCK Conditions

- read-first boot route was skipped;
- evidence references are missing for critical claims;
- generic pass wording appears;
- production/autonomy claim appears in bounded task.

## WARN Conditions

- evidence exists but operational delta is weak;
- part of checks not executed but explicitly marked not-run.

## Not-Proven Boundary

- complete Important Six;
- Astronomicon/Inquisition TUI;
- production autonomy.
