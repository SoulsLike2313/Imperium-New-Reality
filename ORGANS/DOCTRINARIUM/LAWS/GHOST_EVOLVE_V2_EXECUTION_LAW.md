# Ghost_Evolve V2 Execution Law — V0.4 Candidate

Status: `CANDIDATE_NOT_CANON`
Owner candidate: `Doctrinarium` + `Inquisition` + `Mechanicus`

## Law
Ghost_Evolve V2 means every task must leave IMPERIUM stronger, not only produce a one-time answer.

## Principles
- Servitor enters IMPERIUM body, not isolated task text.
- Servitor asks questions through relevant organs before acting.
- Agent reasoning and local system capability must be separated.
- Script-first and replayable residue is preferred over one-time reasoning.
- Every task must include a red-team/self-inquisition phase before verdict.
- Verdicts downgrade on stale, dirty, contradictory, unreplayable or unowned claims.
- Every task leaves reusable artifacts: matrices, scripts, validators, lessons, receipts or capability gaps.

## Mandatory capability split
Every claim must be tagged as one of:

`LOCAL_SCRIPT_FIRST`, `LOCAL_MANUAL_COMMAND`, `CANDIDATE_SCRIPT_FIRST`, `AGENT_REASONING_ONLY`, `EXTERNAL_RESEARCH`, `OWNER_MANUAL_CONFIRMATION`, `FUTURE_CAPABILITY_GAP`

## Minimal task phases
| Phase | Meaning |
| --- | --- |
| ROLE_ENTRY_ACK | Read Officio/role/language/response contract and declare authority. |
| EVIDENCE_BOUNDARY | Declare HEAD, files, inputs, missing evidence, dirty state. |
| ORGAN_QUESTIONING | Ask task questions through relevant organ matrices. |
| BUILD_OR_ANALYSIS | Do the task without claiming unsupported capability. |
| CAPABILITY_SPLIT | Classify what is script-first/manual/agent-only/future gap. |
| RED_TEAM_RESCAN | Attack the result before final verdict. |
| VERDICT_AND_RESIDUE | Emit PASS/WARN/BLOCK and reusable residue. |

## Forbidden claims
- `manual_agent_reasoning_as_system_capability`
- `runtime_pass_without_replay_receipt`
- `clean_state_without_git_status_or_equivalent`
- `canon_claim_without_owner/canon_admission`
