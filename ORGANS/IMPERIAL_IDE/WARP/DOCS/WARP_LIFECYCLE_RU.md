# Жизненный цикл WARP-сессии

```
INTAKE → PLAN → BUILD → VALIDATE → GATE → RELEASED
                                       ├→ HELD       (доработка, остаётся в WARP)
                                       └→ DISCARDED  (отбракован, ядро чисто)
```

| Стадия | Что происходит |
|--------|----------------|
| **INTAKE** | Сессия открыта (авто или кнопкой). Регистрируются участники: органы, Сервиторы, ядро (read), API, CLI. |
| **PLAN** | Задаётся план задачи и критерии (definition-of-done) с минимальным уровнем доказательств. |
| **BUILD** | Работа. Все записи — через `write_artifact()` в workdir. Каждая запись стейджит диф и зону. Ядро НЕ трогается. |
| **VALIDATE** | Собираются метрики с уровнем доказательств (E1…E6). `record_metric()`. |
| **GATE** | `run_gate()` → RELEASE / HOLD / DISCARD. Дирижёр всех правил. |
| **RELEASED** | Шлюз разрешил. Выходит `release_manifest.json` (план продвижения). Ядро НЕ тронуто — ждёт одобрения. |
| **HELD** | Что-то не выполнено, но исправимо. Продукт остаётся в WARP на доработку. |
| **DISCARDED** | Отбракован (напр. fake-green). Artifacts стираются, остаётся расписка (receipt). |

## Вердикты шлюза
- **RELEASE** — все обязательные критерии выполнены на уровне доказательств ≥ порог.
- **HOLD** — обязательный критерий не выполнен/нет доказательства, но фиксится.
- **DISCARD** — нарушение целостности (fake-green: «pass» без доказательств).

## Артефакты каждой сессии
```
WARP/runtime/<session>/
  events.jsonl            # append-only летопись всего
  session_manifest.json   # участники/план/критерии/метрики/дифы/вердикт
  artifacts/              # всё, что собрано (продукт + отходы)
  release_manifest.json   # если RELEASE: план продвижения (pending owner)
  discard_receipt.json    # если DISCARD: расписка об отбраковке
```
