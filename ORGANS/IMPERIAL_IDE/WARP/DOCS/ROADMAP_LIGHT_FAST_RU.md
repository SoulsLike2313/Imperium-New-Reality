# Прицениваюсь: что сделать, чтобы всё стало лёгким и быстрым механизмом

Цель: чтобы ты уже мог задавать задачи, собирать доказательства и видеть работу
в IDE и WARP. Ниже — честная оценка по этапам, от «сразу» до «позже».

## Этап 0 — сегодня (30–60 мин, без кода ядра)
- [ ] Распаковать этот пакет в `ORGANS/IMPERIAL_IDE/WARP/`.
- [ ] Добавить `WARP/runtime/` в `.gitignore`.
- [ ] Выставить env: `IMPERIUM_ROOT=E:\IMPERIUM_NEW_GENERATION_NEW_REALITY`,
      `WARP_ROOT=...\WARP\runtime`.
- [ ] Прогнать `python warp_smoke.py` — увидеть DISCARD и RELEASE пути.
- [ ] Открыть TUI: `python UI/warp_tui.py --smoke`.

## Этап 1 — «могу задавать задачи» (1–2 дня)
- [ ] Встроить `WarpPanel` в Imperial IDE Workbench как панель `13 ⊕ WARP`.
- [ ] Повесить авто-хук `open_for_task` на старт любой задачи IDE/CLI.
- [ ] Шаблон задачи: копируешь `SAMPLE/sample_task.json`, меняешь поля.
- [ ] Первая реальная задача вида THIRD_PARTY (безопасно, ядро не трогаем).

## Этап 2 — «вижу работу и доказательства» (3–5 дней)
- [ ] Подключить Механикус-бридж: вызовы инструментов пишут метрики в сессию.
- [ ] Авто-сбор E3-доказательств из логов выполнения (не ручные value).
- [ ] Подсветить дифы по зонам в GUI (CORE=красный, UNKNOWN=амбра).
- [ ] Проверка fake-green на живой задаче (должен быть DISCARD).

## Этап 3 — «жёсткий релиз» (1 неделя)
- [ ] Связать RELEASE с MOVE_DELETE_APPROVAL_GATE (одобрение владельца).
- [ ] Промоут RELEASED-артефакта в ядро через move_receipt (move-before-delete).
- [ ] Подключить CONTINUITY_GATE: сверка events.jsonl с git состоянием.

## Общие правки для «лёгкости»
- Держи ядро read-only в рантайме — вся запись только в `artifacts/`.
- Старт сессии < 1 сек (stdlib, без зависимостей) — проверено в smoke.
- Одна сессия = одна папка. Чистить старые рантаймы по расписанию.
- Не плоди форматы: всё в JSON/JSONL, читается jq и глазами.
- Не доверяй само-декларациям (E2) — только исполнение (E3+).

## Чего СЕЙЧАС НЕТ (честно)
- Нет реального исполнения инструментов — бридж по умолчанию dry-run.
- GUI/PowerShell не запускались в Linux-сандбоксе — проверь на своём PC.
- Промоут в ядро пока ручной (по одобрению), автоматизация — Этап 3.
