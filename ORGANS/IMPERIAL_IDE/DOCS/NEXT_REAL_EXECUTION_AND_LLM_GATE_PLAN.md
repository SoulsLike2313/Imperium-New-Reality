# Next Real Execution and LLM Gate Plan

Real tools and live LLM backends remain blocked. A future task must provide:

1. Owner-approved per-tool allowlist and argument schemas.
2. Process sandbox and timeout/resource limits.
3. Secret provider contract with no credentials in repository files.
4. Preflight, invocation, and post-run receipts.
5. Explicit one-shot execution before any persistent runtime is considered.
6. Live token and cost measurement separated from candidate estimates.
7. Inquisition negative tests for shell injection, path escape, and fake-green.
8. Administratum release evidence and owner rollback procedure.

Until those gates exist, Workbench capsules, WARP promotion, and MetaOS model
runners stay dry-run, manifest-only, or deterministic-stub implementations.
