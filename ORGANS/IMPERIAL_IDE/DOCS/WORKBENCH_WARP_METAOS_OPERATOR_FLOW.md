# Workbench, WARP, and MetaOS Operator Flow

1. Enter through the Imperial IDE control shell.
2. Inspect governance, current task, reports, and Mechanicus policy.
3. Open Workbench as the read-only owner surface.
4. Create WARP sessions explicitly when isolated runtime work is needed.
5. Run MetaOS deterministic smoke for route, servitor, chronicle, and gate models.
6. Send tool requests through Mechanicus dry-run adapters.
7. Send complete evidence to the Administratum bundle gate.
8. Treat `RELEASED` as permission to create a release manifest only.

No step automatically writes to the kernel, invokes an external LLM, starts a
persistent worker, or enables arbitrary shell execution.
