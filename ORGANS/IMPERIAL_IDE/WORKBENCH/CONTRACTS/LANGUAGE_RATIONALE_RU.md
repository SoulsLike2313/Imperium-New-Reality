# Зачем разные языки в одном приложении — и кто за что отвечает

Это не «зоопарк ради зоопарка». Каждый язык закрывает свою задачу там, где он
объективно сильнее, а склейка идёт через простые файловые контракты (JSON/JSONL),
чтобы слои не зависели друг от друга и не падали вместе.

| Слой | Язык | Почему именно он | Что делает |
|------|------|------------------|------------|
| **GUI (основной)** | **Python + Tkinter** | Ноль установки: Tkinter идёт в стандартной поставке Python, который у тебя уже есть для органов. Запускается двойным кликом, не тянет npm/Electron. | Главное окно: nebula-шапка, панели органов, стойка капсул Сервиторов, лог-консоль, диспетчер задач. |
| **TUI** | **Python (stdlib, ANSI 256)** | Нужен «голый» терминальный пульт без зависимостей — заходит по SSH, в любой консоли Windows Terminal/conhost. ANSI-цвета и box-drawing рисуют картинку кодом. | Текстовый интерфейс того же пульта: статус моста, органы, капсулы. Есть режим `--smoke` для самопроверки. |
| **Капсулы Сервиторов** | **Python** | Один процесс = одна сессия Сервитора. Файловая очередь (JSONL) — дёшево, переживает перезапуск, читается глазами. | `servitor_capsule.py` — берёт задачи из очереди, отдаёт по CLI, пишет результаты и heartbeat. По умолчанию dry-run. |
| **Супервизор капсул** | **PowerShell** | Родная оркестрация процессов в Windows: запуск, контроль живости, рестарт, fan-out. Идёт в коробке ОС. | `servitor_capsule_supervisor.ps1` — поднимает 2–3 капсулы, держит их живыми, раздаёт задачи. |
| **Лаунчеры** | **PowerShell + cmd** | `.ps1` — основной (детект Python, выбор поверхности, флаги). `.cmd` — fallback, если политика выполнения PowerShell заблокирована. | `run_imperial_workbench.ps1` / `.cmd`. |
| **Токены темы** | **TypeScript** | Один типизированный источник правды для палитры/статусов. Любой будущий web/Electron слой импортирует его, а Python и C# читают JSON-двойник — слои не разъезжаются по цветам. | `THEME/imperium_theme.ts` → `imperium_theme.json`. |
| **Мост в Механикус** | **Python** | Должен звать уже существующий органный мост (он на Python) и держать ту же границу безопасности. | `BRIDGES/mechanicus_workbench_bridge.py`. |
| **Нативная оболочка (опц.)** | **C# / WPF** | Если захочешь компилируемый `.exe` с GPU-рендером (blur/glow/векторные сигилы) и без видимого Python. Не обязателен — Python GUI самодостаточен. | `NATIVE_CSHARP/` — собирается `dotnet publish`. |

## Принцип склейки
- Слои общаются **через файлы-контракты**, а не через прямые вызовы: очереди задач (`*.tasks.jsonl`), результаты (`*.results.jsonl`), heartbeat (`*.state.json`), тема (`imperium_theme.json`).
- Поэтому GUI, TUI, супервизор и нативный слой можно запускать по отдельности, и падение одного не роняет остальные.
- **Единая граница безопасности:** реальное выполнение выключено по умолчанию во всех слоях. Включается только явным флагом и только через органный мост Механикуса.
