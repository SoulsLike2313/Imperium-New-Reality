# Mechanicus Arsenal Mass Intake Policy V0.1

Purpose: know and structure many possible capabilities without pretending they are installed/trusted.

Laws:
1. List broadly, validate narrowly.
2. No CANON without receipt/evidence.
3. No tool install in mass intake.
4. External tools begin as CANDIDATE unless locally proven.
5. Internal repo practices may become SANDBOX/CANON only with evidence paths.
6. Capability folders must include card, README, validation plan, usage contract, risks.
7. LLM/cloud adapter sections are reserved for later dedicated work.
8. Owner questions must be recorded instead of guessed.
9. Mechanicus owns tools/checkers/validators.
10. Inquisition can later audit dirt/fake-green; do not create broad cleanup now.
