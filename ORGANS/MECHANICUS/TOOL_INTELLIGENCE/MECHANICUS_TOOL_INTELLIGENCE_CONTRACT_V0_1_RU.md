# Mechanicus Tool Intelligence Contract V0.1 / Контракт Разума Инструментов

## Purpose / Назначение

Mechanicus becomes the engineering judgement organ of Imperium New Reality. It does not merely list commands. It evaluates a task, chooses fitting languages/tools/libraries/algorithms, explains the decision, registers the tool, and exposes the result to Data Atlas.

Mechanicus отвечает не только за список команд. Он должен оценивать задачу, выбирать подходящие языки, инструменты, библиотеки и алгоритмы, объяснять выбор, регистрировать инструмент и делать его видимым в Data Atlas.

## New law / Новый закон

```text
NO TOOL WITHOUT PASSPORT.
NO SCRIPT WITHOUT OWNER.
NO LANGUAGE WITHOUT REASON.
NO PATCH WITHOUT MECHANICUS REGISTRATION.
NO DATA WITHOUT ATLAS VISIBILITY.
```

## Work mode / Режим работы

For the current development phase the project intentionally focuses on two organs:

1. **Mechanicus** — tool intelligence, language selection, capability registry, validation recipes.
2. **Data Atlas** — visibility, ownership, passports, dirt, lifecycle, cleanup lanes.

Every future implementation patch should be accompanied by a Mechanicus/Data Atlas registration layer.

## What Mechanicus must eventually answer / На что Mechanicus должен отвечать

For any task Mechanicus should produce:

- task class;
- constraints and risks;
- recommended language;
- recommended tools and libraries;
- algorithmic approach;
- validation route;
- evidence output route;
- Data Atlas visibility impact;
- fallback options;
- why not use other candidate languages/tools.

## First foundation scope / Состав первого основания

This V0.1 foundation introduces:

- language registry;
- tool passport schema;
- seeded tool passport registry;
- task assessment template;
- research queue;
- Data Atlas coupling doctrine;
- operational power auditor skeleton.

It does not yet make autonomous decisions. The next patch should generate the first real Mechanicus reports from this foundation.
