# OWNER VISUAL CONTRACT (RU)

## Что требуется сейчас

Нынешний результат уже лучше, но нужен еще один шаг:
1. Сильнее приблизить форму к референсу.
2. Накинуть анимаций, которые просто показывают “жизнь органа”, но не врут про backend activity.
3. Усилить нейро sci-fi слой, потому что Механикус внутри Sanctum — это нейронная зона мозга, отвечающая за память инструментов и механизмов.

## Какой должен быть общий образ

Не просто forge-техно-панель.  
Это **нейро-кузница памяти инструментов**.

Должно читаться одновременно:
- Mechanicus / forge / machine intelligence
- neuro / synapse / cognition / second-brain
- operator capsule / organ panel
- хранилище инструментов, скриптов, валидаторов, механизмов

## На что давим визуально

### 1. Ближе к референсу
Надо сильнее подтянуть:
- форму секций;
- пропорции;
- shell language;
- плотность иерархии;
- серьезность “машинной капсулы”.

### 2. Жизнь органа
Добавить мягкие ambient-анимации:
- медленный pulse контуров;
- очень тонкие бегущие токи/энергия;
- нейронные соединения/схемные вспышки;
- микроскан линии;
- плавающая “живость” в фоне;
- мягкое свечение секций;
- subtle reactive hover/micro motion.

Но:
- это не должно симулировать ложную рабочую активность;
- нельзя подменять реальную работу красивой анимацией;
- анимации должны быть “atmospheric / organ-life”, а не “fake telemetry”.

### 3. Нейро слой
Нужно ощутить, что это часть мозга:
- тонкие synapse-like traces;
- faint neural map / mesh / memory lattice;
- схемные или нейронные нити;
- ощущение памяти инструментария;
- возможно отдельный memory field / cognition band / neural current.

## Что именно можно добавить

Можно добавить:
- subtle animated background grid / depth haze;
- neural line glow layer;
- pulse around sigil / mission core;
- low-frequency breathing light in panel edges;
- tiny moving particles / charge nodes;
- animated ambient forge ember / blue energy mix;
- restrained animated separators / progress-like carriers without fake task semantics.

## Жесткие запреты

1. Нельзя превращать это в игрушку или кислотный sci-fi.
2. Нельзя терять читаемость.
3. Нельзя спамить анимацией.
4. Нельзя делать тяжелые или лагающие эффекты.
5. Нельзя привязывать fake animation к PASS/BLOCK/CONNECTED.
6. Нельзя разрушать truth discipline.
7. Нельзя снова скатиться в generic dashboard.

## Acceptance по Owner-смыслу

Owner должен открыть результат и почувствовать:
“Да, это уже реально империумная нейро-механическая зона мозга, а не просто хороший темный дэшборд.”
