# INQUISITION Servitor Contract V0.1

## Input Contract

Servitor query must include:
- `task_id`
- `question`
- optional `mode`

## Output Contract

Return `newgen_organ_verdict_v0_1` with:
- anti-fake-green verdict;
- applied audit rules;
- required anti-dirty/duplicate actions;
- forbidden shortcuts;
- evidence requirements and not-proven boundary.

## BLOCK Conditions

- PASS claim without evidence refs;
- unresolved dirty work;
- unresolved duplicate drift;
- semantic contradiction/readability collapse;
- skipped hard audit.

## WARN Conditions

- partial audit execution explicitly marked as NOT_RUN and non-PASS critical claims.

## Not-Proven Boundary

- autonomous self-healing of all repository drift;
- full semantic intelligence;
- production autonomy.
