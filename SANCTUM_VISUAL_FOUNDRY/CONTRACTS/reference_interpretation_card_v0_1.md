# Reference Interpretation Card V0.1

## What the reference means functionally

1. The left/center visual area must represent the living IMPERIUM neural shell, not a passive wallpaper.
2. Right panel must represent the currently active organ context and show operationally meaningful blocks.
3. Top strip must expose truth context (scope/head/visual state) with compact readability.
4. Raw technical data is accessible but remains secondary.

## What the reference means stylistically

1. Core style mix: 60% black/red Mechanicus forge, 20% dark-purple nebula metallic, 20% clean sci-fi glass/holograph.
2. Visual density target is medium-high (5-7/10) with clear grouping and hierarchy.
3. Motion should communicate life (neural pulse and zone emphasis), not decorative noise.

## What must not be copied literally

1. Do not imitate any single screenshot pixel-by-pixel.
2. Do not fake live values as real backend truth.
3. Do not turn raw terminal output into dominant visual surface.

## Translation into IMPERIUM form

1. Split-screen architecture: brain/zones left and active organ panel right.
2. Clickable organ nodes update the right context panel.
3. Tool registry, contour readiness, and Sanctum-level actions are explicit operational blocks.
4. Raw mode is toggle-based and hidden by default.
