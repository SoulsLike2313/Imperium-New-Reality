## What Mechanicus is ready to do now

Mechanicus can support script-first local checks that use current paths and standard-library dependencies, register cross-organ tools, preserve receipts, classify capability status, and run read-only repository validation. The most reliable current area is core-shape and post-work validation.

## What Mechanicus must become next

Mechanicus should become the current-root executable control plane for local tools, not an archive of cards. The next maturity step is a registry plus validator that rejects missing/stale paths, distinguishes direct tools from delegated cross-organ tools, records replay evidence, and defaults to PC-local read-only execution.

Recommended status: `READY_FOR_LOCAL_SCRIPT_FIRST_HARDENING_WITH_WARNINGS`.
