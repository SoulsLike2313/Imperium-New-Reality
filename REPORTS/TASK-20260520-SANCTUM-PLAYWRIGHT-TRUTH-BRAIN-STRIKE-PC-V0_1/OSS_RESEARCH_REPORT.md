# OSS Research and Admission Report

## Research Coverage
Assessed candidate tools and patterns across required categories:
- graph visualization
- terminal embedding
- animation
- performance optimization
- UI composition
- visual regression
- truth binding / event transport

## Near-Term Admissions (NOW)
1. D3 Force Simulation
- Use for dynamic neural topology and zone-link orchestration.
2. Motion
- Use for controlled reveal/pulse transitions with bounded animation cost.
3. Playwright Visual Comparisons
- Add snapshot baseline checks for anti-regression and anti-fake-green.
4. Server-Sent Events
- Add low-complexity telemetry push channel from backend to Sanctum.
5. requestAnimationFrame performance probes
- Keep periodic smoothness checks in local and CI quality gates.

## Medium-Term Path (LATER)
1. xterm.js
- Upgrade LIVE terminal from static entries to full terminal UX.
2. Sigma.js or Cytoscape.js
- Move to scalable graph engine when node/edge count and interactions increase.
3. Lit componentization
- Split Sanctum into independently testable UI modules.
4. WebSocket transport
- Introduce bidirectional channels for remote orchestration control.

## Rejected for Current Strike
- htmx: good for hypermedia forms and partial swaps, but not a strong fit for this graph-heavy brain UI target.

## Future Brain UI Enhancement Path
1. Replace static layout constants with force-directed runtime graph (D3).
2. Add zone-level signal health streams over SSE.
3. Add sigma/cytoscape rendering mode toggle for large-map explorations.
4. Add Playwright visual baseline gate in pre-merge pipeline.
