# Playwright Truth Audit Requirements

Mandatory checks:

## A. Boot / health
- server launches
- UI opens
- top bar loads
- health/status endpoints respond

## B. Visible claims
Check and report:
- server status
- API status
- HEAD value
- worktree state
- active organ
- connected/placeholder/locked counts
- warning/error/block counts
- latest report path
- latest screenshot path
- latest evidence path

## C. Action checks
Minimum click/check set:
- refresh state
- mechanicus visual-status
- mechanicus visual-tools
- mechanicus visual-check
- mechanicus visual-identity
- report path action
- screenshots path action
- API JSON view action

## D. Truth checks
Verify that what UI claims matches:
- actual files/paths
- actual backend JSON
- actual receipts/reports
- actual connected/placeholder/locked reality

## E. Screenshots
Must capture:
- overview
- active brain view
- mechanicus expanded view
- right truth panel
- failure/gap examples
- action-click result examples

## F. Output
Produce machine-readable summary and human-readable audit.
