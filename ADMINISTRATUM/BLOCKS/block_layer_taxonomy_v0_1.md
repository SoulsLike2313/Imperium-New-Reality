# Administratum Block Layer Taxonomy V0.1

This is a seed taxonomy for future Block Spine compatibility.
It does not implement global Block Spine behavior.

## Layer Model

- `L0` System Blocks
- `L1` Organ Blocks
- `L2` Macro Blocks (`CAPABILITY`, `FOUNDATION`)
- `L3` Micro Blocks
- `L4` Leaf Files

## Administratum Organ Block (Seed)

`ADMINISTRATUM_ORGAN_BLOCK`

Macro blocks:
- `ADMINISTRATUM_CAPABILITY_BLOCK`
- `ADMINISTRATUM_FOUNDATION_BLOCK`

### Capability Block (What Administratum can do)

- maintain registration card schemas;
- track current accepted head/points;
- register active organs and pending next tasks;
- register future-compatible WARP/CLI card formats.

### Foundation Block (Why Administratum can do it)

- registration contract;
- ownership-separation law;
- addressability conventions;
- chronology and current-truth update rules.

## Out of Scope

- no cross-repo block migration;
- no runtime routing engine;
- no IDE/WARP activation.
