# Administratum Registration Contract V0.1

## Purpose

Administratum records accepted truth and artifact addresses.
Administratum does not claim real ownership over foreign organ artifacts.

## Registration Law

For each registered card/artifact:
- `real_owner_organ` = organ that owns implementation and lifecycle.
- `registration_owner_organ` = `ADMINISTRATUM`.
- `validation_owner_organ` = organ that validates behavior or truth contract.

## Registration Objects

Administratum registers:
- organ cards;
- accepted points;
- current head card;
- active organ index;
- pending next task index;
- artifact index;
- block taxonomy seed;
- future-compatible schema seeds (task session, WARP session, CLI worker report).

## Scope Boundary

This contract does not authorize:
- WARP runtime implementation;
- CLI Worker implementation;
- Agent IDE implementation;
- Inquisition/Astronomicon implementation;
- Mechanicus or Officio refactors.

## Truth Rule

PASS claims require evidence paths and checkable receipts.
No fake dashboard controls or fake action affordances are allowed.
